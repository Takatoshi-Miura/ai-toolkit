# 分析・アドバイスガイド

fetch.py の出力JSONをもとに、Claudeが分析・アドバイスを生成するための指針。

---

## 聖遺物ビルド評価の観点

### メインオプション適正チェック

| 部位 | 推奨メイン（汎用） |
|---|---|
| 花 | HP固定 |
| 羽 | 攻撃力固定 |
| サンド | 攻撃%、元素熟知、元素チャージ効率（キャラ依存） |
| ゴブレット | 元素ダメ%（物理DPSは物理ダメ%）、HP%（耐久・ヒーラー） |
| サークレット | 会心率、会心ダメージ（優先）、元素熟知、回復効果（ヒーラー） |

### サブオプション優先度

**攻撃/DPS系**: 会心率 > 会心ダメージ > 攻撃% > 元素熟知 > チャージ効率
**耐久/ヒーラー系**: HP% > 会心率/ダメ > チャージ効率

### セット効果の評価

- 4セット揃いが強力なキャラ（胡桃、雷電将軍など）→ セット優先
- サブオプション質が高ければ2+2も検討
- 初心者は4セット、上級者はサブオプション次第で2+2も選択肢

### レベルの確認

- `artifacts[].score` はサブオプションスコア（+20前提）
- 実際のレベルは `reliquary.level`（最大21 = +20）
- +20未満なら「強化を優先」とアドバイス

---

## スキルレベルの評価

`skill_levels` の配列（通常攻撃 / 元素スキル / 元素爆発 の順）を確認。

- 元素スキル・元素爆発を優先して上げる（8〜10が目標）
- 通常攻撃は物理DPSキャラのみ優先
- 10/10/10 が理想だが、9/9/9 でも実用上問題なし

## パーティ編成の観点

`cmd_party` の出力をもとに提案する。

### 元素反応シナジー

| 反応名 | 組み合わせ | 特徴 |
|---|---|---|
| 蒸発 | 水 + 炎 | ダメ1.5〜2倍 |
| 溶解 | 氷 + 炎 | ダメ1.5〜2倍 |
| 感電 | 水 + 雷 | 継続ダメ |
| 超開花 | 草 + 水 + 雷 | 種ダメ（元素熟知依存） |
| 拡散 | 風 + 任意 | デバフ拡散 |
| 超激化 | 草 + 雷 | 雷ダメ強化 |
| 開花 | 草 + 水 | 種生成（元素熟知依存） |

### 役割バランス

メインDPS / サブDPS / サポート / ヒーラーの4枠が基本。全員のキャラリストから：
1. 最もスコアが高く主力になれるキャラをメインDPS候補に
2. 元素シナジーを考慮してサブDPS・サポートを選択
3. ヒーラーまたはシールダーを1枠確保

## エラー時の対応

| エラー | 原因 | 対処 |
|---|---|---|
| `UIDが不正` | UIDの桁数・形式が間違い | UID確認を依頼 |
| `UIDが見つかりません` | 存在しないUID | UID確認を依頼 |
| `メンテナンス中` | ゲームサーバーメンテ | 時間をおいて再試行 |
| `レート制限` | APIアクセス過多 | 数分待って再試行 |
| `キャラ情報なし` | プロフィール非公開 | ゲーム内設定で公開を案内 |
| `データ取得失敗`（status） | Cookie期限切れ・不正 | Cookie再取得を案内 |
