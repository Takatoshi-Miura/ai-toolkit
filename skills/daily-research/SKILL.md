# デイリーリサーチ

日本語で回答すること。

## 実行手順

1. [TOPICS.md](TOPICS.md) を読み込み、収集対象トピックと各トピックの出力形式を取得する
2. 出力先ディレクトリを準備する: `mkdir -p ~/Downloads/daily-research/`
3. **収集済みURLリスト**を空のリストとして初期化する（重複排除に使用）
4. 実行日の年月（`YYYY年M月`）を取得し、**当月フィルター**として保持する
5. 各トピックについてWebSearchで記事を収集し、各記事のURLをWebFetchで取得して内容を把握する（直接取得URLがあればWebFetch優先）
   - 収集した各記事の**公開日を確認**し、当月（実行月）以外の記事はスキップする
   - 収集した各記事のURLを収集済みURLリストと照合し、**既出のURLはスキップ**する
   - 新規URLのみ収集済みURLリストに追加し、そのトピックの記事として採用する
   - スキップにより件数が足りない場合は、検索クエリに `{YYYY}年{M}月` を追加して再検索し不足分を補う
6. 共通ヘッダー + 目次を出力し、各トピックはTOPICS.mdに定義された出力形式テンプレートに従って整形する
7. `~/Downloads/daily-research/digest-YYYY-MM-DD-HHMM.md` に出力する
8. 収集結果サマリー（成功/失敗トピック数、総記事数、重複スキップ数、出力パス）を報告する

## 制約

- 単一トピックの失敗で全体を停止しない。収集できた分だけで出力する
- WebFetch失敗時はWebSearchにフォールバックする
- ファイル名に時刻を含むため、同日複数回実行すると別ファイルとして出力される
- トピック追加時は [TOPIC-TEMPLATE.md](TOPIC-TEMPLATE.md) をベースに使用する
- **当月（実行月）以外の記事は掲載しない**。公開日が確認できない場合は記事本文の日付情報から推定し、それでも判断できない場合はスキップする
- **同一URLは複数トピックに重複して掲載しない**。先に収集したトピックを優先し、後のトピックではその記事をスキップする
- **サブエージェント（Taskツール）は使用しない**。全ての収集・処理を単一のコンテキスト内で完結させる
