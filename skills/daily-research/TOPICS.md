# 収集トピック定義

daily-researchスキルが収集するトピックを管理するファイルです。
トピックの追加・削除はセクションを追加・削除するだけで反映されます。
新規追加時は [TOPIC-TEMPLATE.md](TOPIC-TEMPLATE.md) をコピーして使用してください。

---

## 共通ヘッダー

```
# Daily Research - {YYYY-MM-DD}

> 収集日時: {YYYY-MM-DD HH:MM}
> 収集トピック数: {topic_count}
> 総記事数: {total_articles}

```

## 1. Claude Code 更新情報

| 項目 | 値 |
|------|-----|
| カテゴリ | tech |
| 直接取得URL | https://code.claude.com/docs/ja/changelog |
| 言語 | ja |
| 優先度 | high |

### 収集手順

1. changelogページをWebFetchで取得し、**最新バージョン**のセクションのみを抽出する
2. 最新バージョンの更新内容から**機能追加・改善のみ**をピックアップする（バグ修正は除外）
3. 各機能について、技術ブログや公式ドキュメント等をWebSearchで調査し、活用事例や詳細情報を収集する

### 出力形式

```
## [Tech] Claude Code 更新情報（{バージョン名}）

### {機能名}
- **概要**: {何が追加・変更されたかを平易な日本語で説明}
- **活用事例**: {技術ブログ等から収集した具体的な使い方や事例。見つからなければ想定される活用シーンを記載}
- **活用提案**: {この機能を日常の開発でどう活かせるかの具体的な提案}

...（機能追加ごとに繰り返す）
```
