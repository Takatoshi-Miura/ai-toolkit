## Phase 1: 問題の定義（What）

### 概要

あるべき姿と現状とのギャップにより、問題を明確にする。

### 実行手順

#### 1-1. 現状を把握する

**基本原則**:
- 事実と意見を切り分けて、事実を把握する
- 語尾は「〜いる」「〜である」という状態で記載する
- 事実に基づく情報（可能なら一次情報）を根拠に記載する

**NG例とOK例**:
- NG例：研修を実施している ← 状態ではなく活動・施策になっておりNG
- OK例：社員の技術力が不足している ← 状態を表現しているためOK

**5つのポイント**:
1. **ゼロベース思考**で、現地現物による一次情報で現状を把握する
2. **「いつ」を把握する**: 発生時期、頻度
3. **「どこで」を把握する**: 発生場所、範囲
4. **「どのように」を把握する**: 発生方法、メカニズム
5. **「どの程度／頻度で」を把握する**: 発生頻度、影響範囲

**情報源の優先順位**:
- 一次情報を重視する（現場確認、当事者ヒアリング）
- 二次情報を用いる際は根拠となる裏付けを取る

#### 1-2. あるべき姿を明確にする

**基本原則**:
- あるべき姿、現状、ギャップは時系列や概念を揃え、対になるようにする
- あるべき姿と現状は、比較表現ではなく絶対値で表現する
- 可能なら定量的に記載する

**あるべき姿を描くための4つの視点**:

1. **目的に立ち返って考える**
   - そもそも何のためにやっているのか、原点から考え直してみる
   - 例：「売上向上のため」→「顧客満足度向上のため」→「顧客の課題解決のため」

2. **時間軸を伸ばして考える**
   - 将来から発想したら、何か見えてこないか
   - 例：1年後、3年後、5年後にどうなっているべきか

3. **立場を上げて考える**
   - 上位者といった一段高い視点で考え直してみる
   - 例：担当者→リーダー→マネージャー→経営者の視点

4. **逆の視点から考える**
   - 反対者の立場で考えたり、裏から物事を見てみる
   - 例：顧客、競合、社外の視点

**定量化のポイント**:
- 「品質」や「状態」を「時間」「回数」「比率」に変換する
  - NG例：品質が高い
  - OK例：不良品率が1%以下である

**比較表現と絶対値の違い**:
- NG例：現状より良い状態
- OK例：月間200台販売している状態

#### 1-3. 問題（ギャップ）を明確にする

**基本原則**:
- 現状とあるべき姿のギャップが問題になる

**OK例**:
- 現状：PCを月間180台販売している
- あるべき姿：PCを月間200台販売している
- 問題：PCを月間20台販売できていない

**NG例**:
- 現状：売上が低い ← 比較表現で定量化されていない
- あるべき姿：売上を上げる ← 比較表現で定量化されていない
- 問題：売上が不足している ← ギャップの大きさが不明

- [ ] 語尾は「〜いる」「〜である」という状態で記載できているか
- [ ] 現状が状態ではなく活動・施策になっていないか
- [ ] あるべき姿と現状が比較表現ではなく絶対値で表現できているか
- [ ] あるべき姿、現状、ギャップは時系列や概念を揃え、対になっているか
- [ ] 可能な限り定量化しているか
- [ ] ゼロベース思考で考えているか

## Phase 2: 所在の特定（Where）

### 概要

問題をブレイクダウンして所在を特定する。

### 実行手順

#### 2-1. 問題をピラミッド構造に整理する

**基本原則**:
- 問題をMECEに分解する
- 事実を必ず見る（事実と意見と混在しない）
- 深掘りする／しないところのメリハリをつける

**MECE分解の方法**:
1. **AとNot Aに切り分ける**
   - 例：国内と海外、大手企業と大手企業以外
2. **数式で分ける**
   - 例：（売上）＝（社員数）×（社員1人あたりの売上高）
3. **プロセスで分ける**
   - 例：営業プロセス：訪問→商談→見積もり→成約
4. **一般的なフレームワークを用いる**
   - 例：3C・4P・AIDMA

**重要な注意点**:
- 問題の分解方法は1通りではないため、問題の所在が明確にならないときは別の分解方法で分解してみる
- 事実に基づいて分解する（推測や意見で分解しない）

**深掘りのメリハリ**:
- すべての階層を均等に深掘りする必要はない
- 問題がありそうな箇所を優先的に深掘りする

#### 2-2. 問題を分解したら、末端の階層に対して問題の有無を考える

**基本原則**:
- 問題がありそうな箇所には「×」、問題ない箇所には「○」をつける
- それぞれに対して、そう判断した理由を記載する

**判定の根拠**:
- 事実に基づいて判定する
- 定量的なデータがある場合はそれを根拠にする
- 推測の場合はその旨を明記する

### チェックリスト

Phase 2を完了する前に、以下を確認してください:

- [ ] 事実に基づいて問題を切り分けられているか
- [ ] 問題の分解方法は適切か（MECE分解できているか）
- [ ] 問題の有無判定（×○）に根拠があるか
- [ ] 深掘りする／しないところのメリハリがついているか


## よくある質問

**Q: 現状とあるべき姿は必ず定量化する必要がありますか？**
A: 可能な限り定量化することを推奨します。ただし、定性的な表現しかできない場合は、できるだけ具体的に記載してください。

**Q: ピラミッド構造は何階層まで分解すればよいですか？**
A: 問題の所在が明確になるまで分解します。一般的には2-4階層程度が多いですが、必要に応じて調整してください。

**Q: すべての末端に×○をつける必要がありますか？**
A: はい。すべての末端の階層に対して問題の有無を判定し、その理由を記載してください。

**Q: Phase 1とPhase 2の違いは何ですか？**
A: Phase 1は問題全体を定義するフェーズです。Phase 2は問題を分解して、どこに問題があるかを特定するフェーズです。
