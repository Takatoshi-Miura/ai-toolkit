# Phase 4: 解決策立案（How）

このファイルはPhase 4の詳細ガイドです。特定フェーズの修正が必要な場合に参照してください。

---

## 概要

課題への解決策を列挙し、評価基準に照らして選択する。

## 実行手順

### 4-1. 解決策となり得ることを幅広く洗い出す

**3つの方法**:

1. **ブレインストーミングで洗い出す**
   - 批判せず、自由にアイデアを出す
   - 量を重視する（質は後で評価する）
   - 他人のアイデアから発想を広げる

2. **ベストプラクティス（成功事例）を参考にする**
   - 同業他社の事例
   - 異業種の成功事例
   - 書籍・論文・ウェブ記事などの情報源

3. **他人の意見を参考にする**
   - チームメンバー
   - 上司・先輩
   - 外部の専門家

**重要な注意点**:
- この段階では評価せず、できるだけ多くの解決策を洗い出す
- 実現可能性を気にせず、自由に発想する
- 最低でも3つ以上の解決策を洗い出すことを推奨

### 4-2. 解決策の評価軸を設定する

**一般的な4つの評価軸**:

1. **効果**: 解決策を実行した時の効果の高さ
   - 課題をどの程度解決できるか
   - 副次的な効果はあるか

2. **コスト**: 解決策を実行するのに必要な投資（人、物、金など）
   - 初期投資コスト
   - 運用コスト
   - 人的リソース

3. **リスク**: 解決策を実行するときの負の影響をもたらす可能性があるか
   - 想定より効果が出ないリスク
   - 副作用が発生するリスク
   - 実行に失敗するリスク

4. **時間**: 解決策の効果が発現するまでの期間
   - 即効性があるか
   - 長期的な効果か

**その他の評価軸の例**:
- 実現可能性
- 持続可能性
- 拡張性
- 組織への影響

**重要な注意点**:
- 評価軸は目的に応じて適切に選択する
- 上記以外の評価軸があれば追加する
- 評価軸は3-5個程度が適切（多すぎると判断が困難）

### 4-3. 評価軸の優先順位を明確にする

**重みづけの方法**:
- 設定した評価軸ごとの重要度を明確にする
- 最も重要な評価軸を第1優先、次に重要な評価軸を第2優先、と順位付ける

**例**:
1. 第1優先：効果（最も重要）
2. 第2優先：時間（早く効果を出したい）
3. 第3優先：コスト（予算に余裕はある）
4. 第4優先：リスク（多少のリスクは許容）

**重要な注意点**:
- 優先順位を明確にすることで、解決策の選択基準が明確になる
- 優先順位は状況に応じて変わる（予算制約がある場合はコストが第1優先になる等）

### 4-4. 洗い出した解決策を評価軸ごとに評価する

**3段階の数値評価**:
- **3**: 良い（評価軸の観点で優れている）
- **2**: 普通（評価軸の観点で標準的）
- **1**: 悪い（評価軸の観点で劣っている）

**評価の注意点**:
- **数値の根拠を明らかにする**: なぜその評価なのかを必ず記載
- **相対評価ではなく絶対評価**: 他の解決策と比較するのではなく、その解決策単体で評価
- **事実に基づいて評価**: 推測や希望的観測ではなく、事実やデータに基づいて評価

**評価の例**:
```
効果: 3
理由：過去の類似施策で30%の改善実績があるため
```

### 4-5. 評価に基づいて解決策を絞り込む

**重要な原則: 評価軸ごとの獲得ポイントは合計しないこと**

なぜ合計してはいけないのか：
- 評価軸の優先順位が反映されない
- 重要な評価軸での低評価が埋もれてしまう

**正しい絞り込み方法**:
1. 第1優先の評価軸で比較する
2. 同点の場合は第2優先の評価軸で比較する
3. さらに同点の場合は第3優先の評価軸で比較する

**例**:
```
評価軸の優先順位：効果 > 時間 > コスト > リスク

A案：効果3、時間2、コスト1、リスク3
B案：効果3、時間3、コスト2、リスク2
C案：効果2、時間3、コスト3、リスク1

選択：B案
理由：第1優先の効果ではA案とB案が同点（3）。
     第2優先の時間ではB案が優れている（3 > 2）。
     よってB案を選択。
```

## チェックリスト

Phase 4を完了する前に、以下を確認してください:

- [ ] 解決策を幅広く洗い出せていること（最低3つ以上）
- [ ] 3つの方法（ブレインストーミング、ベストプラクティス、他人の意見）を活用したか
- [ ] 評価軸の見落としがないこと（効果、コスト、リスク、時間など）
- [ ] 評価軸の優先順位を明確にできていること
- [ ] 解決策を3段階の数字で評価できていること
- [ ] 解決策をそのように評価する理由が明らかにできていること
- [ ] 評価軸の優先順位に基づいて、最も効果的な解決策を絞り込めていること
- [ ] 評価軸ごとの獲得ポイントを合計していないこと

## よくある質問

**Q: 解決策は何個洗い出せばよいですか？**
A: 最低でも3つ以上を推奨します。選択肢が多いほど、最適な解決策を選べる可能性が高まります。

**Q: 評価軸は必ず4つ（効果、コスト、リスク、時間）にする必要がありますか？**
A: いいえ。状況に応じて適切な評価軸を選択してください。3-5個程度が適切です。

**Q: なぜ評価軸ごとの獲得ポイントを合計してはいけないのですか？**
A: 合計すると、重要な評価軸での低評価が埋もれてしまうためです。優先順位に基づいて判断することが重要です。

**Q: すべての評価軸で同点の場合はどうすればよいですか？**
A: その場合は、定性的な判断（実行のしやすさ、組織への影響など）で選択してください。

**Q: 選択された解決策が実現困難な場合はどうすればよいですか？**
A: 評価軸に「実現可能性」を追加して再評価するか、Phase 3に戻って課題を再設定することを検討してください。

## 実例

### 良い例

```markdown
## Phase 4: 解決策立案（How）

- 課題：ヒアリングシートを活用するようにする
- 有効な解決策：B案
- 理由：効果と時間の両方で優れており、早期に高い効果が期待できるため

### 評価軸の優先順位
1. 効果（課題解決に直結するため最も重要）
2. 時間（早く効果を出したい）
3. コスト（予算に余裕はある）
4. リスク（多少のリスクは許容）

### 評価表

|解決策|効果|時間|コスト|リスク|
|---|---|---|---|---|
|A案<br>ヒアリングシートのテンプレートを作成し配布|2<br>理由：活用するかは個人に依存するため|3<br>理由：すぐに実施できる|3<br>理由：コストはほぼゼロ|2<br>理由：活用されない可能性がある|
|B案<br>ヒアリングシート活用を義務化し、上司がチェック|3<br>理由：義務化により確実に活用される|3<br>理由：すぐに実施できる|2<br>理由：上司の工数が増える|3<br>理由：確実に効果が出る|
|C案<br>ヒアリング研修を実施|2<br>理由：研修効果は個人差がある|1<br>理由：研修準備に時間がかかる|1<br>理由：研修費用がかかる|2<br>理由：研修効果が出ない可能性|

※3段階の数値(1~3：数値が大きいほど良い)で評価する
```
