# Phase 3: 原因追及・課題設定（Why）

このファイルはPhase 3の詳細ガイドです。特定フェーズの修正が必要な場合に参照してください。

---

## 概要

原因を深掘りして、真因の裏返しとなる課題を特定する。

## 実行手順

### 3-1. 「なぜ」を5回（複数回）繰り返す

**基本原則**:
- 何がこの問題を引き起こしているのかを分析する
- 「なぜ」を繰り返す回数は目安であり、真因に到達すれば3回でも7回でもよい

**重要な手法: 「広げて」→「絞る」**:
1. **「広げて」**: まず複数の原因候補を網羅的に洗い出す
2. **「絞る」**: その中から最も重要な原因を選択して深掘りする

この手法により、視野を狭めずに的確な分析を行うことができます。

**自責思考に集中する**:
- **自責なもの（集中すべき）**: 自分の行動、自分の意識
  - 例：「自分たちのプロセスが非効率だから」
  - 例：「自分たちのスキルが不足しているから」
- **他責なもの（注力すべきでない）**: 会社、他人、顧客
  - 例：「会社の方針が悪いから」← 自分たちで変えられない
  - 例：「顧客の要求が無理だから」← 自分たちで変えられない

**「変えられない」という思い込みを排除**:
- 一旦網羅的に考えるのは有効だが、他責なものは深掘りしないこと
- 「変えられない」という思い込みで視野が狭まらないように注意する

**フレームワークの活用**:
- **QCD（品質、コスト、納期）**: 複数の観点から原因を考える
  - 品質（Quality）：成果物の質に関する原因
  - コスト（Cost）：費用・リソースに関する原因
  - 納期（Delivery）：時間に関する原因
- **4M（人、機械、方法、材料）**: 網羅的に原因を考える
  - 人（Man）：人的要因
  - 機械（Machine）：設備・ツールの要因
  - 方法（Method）：プロセス・手順の要因
  - 材料（Material）：素材・情報の要因

### 3-2. 真因の裏返しが、解決するべき「課題」になるまで繰り返す

**基本原則**:
- 真因の裏返しが課題になる

**例**:
- 真因：「リーダーがメンバーに仕事を任せていないため」
- 裏返し（課題）：「リーダーがメンバーに仕事を任せるようにする」

**課題の条件**:
- 具体的な行動に落とし込める
- 自分たちで変えられる
- 問題解決に直結する

### 3-3. 真因から逆に見ても「原因と結果」の関係が成り立つかを確認する

**因果関係の双方向性検証**:

原因追及を行った後、逆方向からも因果関係を検証します。

**例**:
```
【順方向】
問題：リーダーが整合性を検討する時間が取れない
↓ なぜ？
原因1：リーダーが他の業務で手一杯のため
↓ なぜ？
原因2（真因）：リーダーがメンバーに仕事を任せていないため
```

**逆方向の検証**:
- 本当にリーダーがメンバーに仕事を任せれば、リーダーに時間の余裕ができるのか？
  - → Yes なら因果関係が成立
- 本当に他の業務に余裕ができれば整合性を検討する時間を取れるか？
  - → Yes なら因果関係が成立

**検証が必要な理由**:
- 因果関係が一方向だけでは、真の原因ではない可能性がある
- 双方向で検証することで、真因の妥当性を確認できる

## チェックリスト

Phase 3を完了する前に、以下を確認してください:

- [ ] 原因追及が自責なもの（自分の行動、自分の意識）に集中できているか
- [ ] 原因追及が他責なもの（会社、他人、顧客）に注力していないか
- [ ] 「広げて」→「絞る」の手法で分析できているか
- [ ] QCDや4Mのフレームワークを活用して網羅的に原因を考えたか
- [ ] 真因の裏返しが、解決するべき「課題」になっているか
- [ ] 課題は具体的な行動に落とし込めるか
- [ ] 真因から逆に見ても「原因と結果」の関係が成り立つか

## よくある質問

**Q: 「なぜ」は必ず5回繰り返す必要がありますか？**
A: いいえ。5回は目安です。真因に到達すれば3回でも7回でも構いません。重要なのは、自責で変えられる真因に到達することです。

**Q: 他責な原因は全く考えてはいけないのですか？**
A: いいえ。「広げて」のフェーズで網羅的に考えるのは有効です。ただし、深掘りする際は自責なものに集中してください。

**Q: 複数の真因がある場合はどうすればよいですか？**
A: 複数の真因がある場合は、それぞれを課題として設定してください。ただし、優先順位をつけることを推奨します。

**Q: 因果関係の双方向性検証で「No」になった場合はどうすればよいですか？**
A: その原因は真因ではない可能性があります。他の原因を探して深掘りし直してください。

**Q: QCDと4Mの両方を使う必要がありますか？**
A: 必須ではありません。目的に応じて適切なフレームワークを選択してください。両方使うこともできます。

## 実例

### 良い例

```markdown
## Phase 3: 原因追及・課題設定（Why）

- [所在の特定]：新規顧客獲得数が目標に達していない
    - L なぜ？：営業活動の量が不足している
        - L なぜ？：訪問件数が少ない
            - ○：深掘りすべきでない（訪問件数は改善傾向にある）
        - L なぜ？：商談化率が低い
            - L なぜ？：ヒアリング不足で顧客ニーズを把握できていない
                - L なぜ？：ヒアリングシートを活用していない
                    - ×：真因（自責で変えられる、双方向で因果関係が成立）

### 課題
- 課題：ヒアリングシートを活用するようにする
- 因果関係の双方向性検証：
  - ヒアリングシートを活用すれば、ヒアリング不足が解消されるか？ → Yes
  - ヒアリング不足が解消されれば、商談化率が向上するか？ → Yes
  - 商談化率が向上すれば、新規顧客獲得数が増えるか？ → Yes
```

### 悪い例（他責に注力している）

```markdown
## Phase 3: 原因追及・課題設定（Why）

- [所在の特定]：新規顧客獲得数が目標に達していない
    - L なぜ？：競合が多い
        - L なぜ？：市場が飽和している
            - ×：真因（市場環境が悪い）← 他責で自分たちでは変えられない

### 課題
- 課題：市場環境を改善する ← 実行不可能な課題
```
