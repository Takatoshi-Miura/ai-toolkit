# コーディングスキル 共通リファレンス

両ワークフロー共通の出力フォーマット・規約を記載する。

---

## Phase完了時の報告フォーマット

各Phase完了時に以下の形式で報告:

```markdown
## Phase X 完了報告

### 実施内容
- （実施した内容を箇条書き）

### 成果物
- （生成されたファイル、URL等）

### 次のPhaseへの引き継ぎ事項
- （次のPhaseで必要な情報）
```

---

## サブエージェント呼び出し規約

### 共通ルール
- promptには具体的な情報を含める（抽象的な指示は避ける）
- エラー発生時は最大3回再試行
- サブエージェントの報告を確認してから次のPhaseに進む

### Plan サブエージェント
- **用途**: 実装計画の立案、コードベース調査
- **subagent_type**: `"Plan"`
- 不明点がある場合はユーザーへの質問を含める

### code-implementer サブエージェント
- **用途**: コード実装、ビルド確認
- **subagent_type**: `"code-implementer"`
- 承認された実装計画とCLAUDE.mdパスを必ず含める

### test-writer サブエージェント
- **用途**: テストコード作成・実行
- **subagent_type**: `"test-writer"`
- テスト対象ファイルパスとフレームワーク情報を含める

---

## 共通の注意事項

- 各Phaseは順序通りに実行（依存関係あり）
- エラー発生時は該当Phaseで停止しユーザーに報告
- Phase完了ごとにTodoWriteで進捗更新
- 過剰な改善提案は避け、要求されたスコープに集中
- 既存コードのスタイルを尊重する
