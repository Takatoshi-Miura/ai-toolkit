# パーソナルコンテキスト対応スキル

## 概要

このスキルは、Google Driveに保存されたパーソナルコンテキストドキュメントを読み取り、ユーザーの特性に合わせた回答を自動的に生成します。技術的意思決定、振り返り、アドバイス依頼など、深い思考を必要とする質問に対して発動します。

## 発動条件

このスキルは以下の状況で自動的に適用されます：

### 発動する場面
- **技術的意思決定・選択の相談**: 「AとBどちらがいいか」「どう実装すべきか」
- **優先順位・方向性の判断**: 「何から始めるべきか」「優先順位は」
- **振り返り・分析・評価**: 「振り返って」「分析して」「評価して」「どう思う」
- **アドバイス・提案の依頼**: 「アドバイスください」「提案して」「どうすればいい」
- **キャリア・学習・成長相談**: 「スキルアップ」「学習方法」「キャリア」
- **自己理解・価値観相談**: 「自分に合った」「価値観」

### 発動しない場面
- 単純なコーディングタスク
- 情報検索・事実確認
- ファイル操作
- デバッグ・エラー修正
- 定型作業

## 手順

### 1. パーソナルコンテキストの取得

Google Driveからパーソナルコンテキストドキュメントを読み取る：

```
ファイルID: 1hDcVtQ5wEz2rPGRrJGK8CspnqSujheAjeZ1PPAj2u6E
ツール: g_drive_read_file(fileId, "docs")
```

以下の情報を抽出：
- **思考スタイル**: 分析型、実験重視、データ駆動型など
- **価値観**: 後悔しない生き方、自己制御感、ミニマリズム、貢献感など
- **専門性**: iOS/Android開発、AIエージェント/MCP開発、元CIVIL等
- **AIへの期待**: キックスターター、自律的実行者、分析アドバイザー
- **コミュニケーション好み**: PREP法、論理的・簡潔、問いかけによる気づき促進
- **避けてほしいこと**: 意図の無視、過度な装飾、非論理的な励まし、一般論のみ

### 2. 質問の意図を理解

ユーザーの質問を以下のカテゴリに分類：
- 技術的意思決定
- 優先順位判断
- 振り返り・分析
- アドバイス・提案
- 学習・成長
- 価値観・自己理解
- その他（複合）

### 3. コンテキストの適用

パーソナルコンテキストを踏まえた回答を構築：

#### 思考スタイルへの対応
- **分析型** → データや具体的な比較を含める
- **実験重視** → 試せるアプローチを複数提示
- **データ駆動** → 客観的な根拠を示す

#### 価値観への対応
- **後悔しない選択** → 長期的な視点での評価軸を提示
- **自己制御感** → 選択肢と裁量を尊重
- **ミニマリズム** → シンプルで本質的なアプローチを優先
- **貢献感** → 成果や影響を明確化

#### 専門性の活用
- **モバイル開発経験** → プラットフォーム固有の考慮事項
- **AI開発経験** → 自動化や効率化の観点

#### AIへの期待に応える
- **キックスターター** → 具体的な叩き台を提示
- **自律実行** → 次のステップを明確化
- **分析アドバイザー** → 多角的な分析と洞察

#### コミュニケーション好みに適合
- **PREP法で構造化**（結論→理由→具体例→結論）
- **論理的・簡潔**（冗長な表現を避ける）
- **問いかけで気づきを促す**

#### 避けるべき要素の排除
- 意図を無視しない
- 過度な装飾・修辞を避ける
- 非論理的な励ましをしない
- 一般論だけでなく具体性を持たせる

### 4. 回答の生成

PREP法に基づいた構造化された回答を提供する。

## 出力形式

```markdown
## 結論
（ユーザーの質問に対する明確な回答・結論を簡潔に2-3行で）

## 理由
（論理的な理由、データや具体例を含めて説明）
- （根拠1）
- （根拠2）
- （根拠3）

## 具体例
（実践可能な具体的アプローチや事例）

### オプション1: [アプローチ名]
- メリット: ...
- デメリット: ...
- 適用場面: ...

### オプション2: [アプローチ名]
- メリット: ...
- デメリット: ...
- 適用場面: ...

## 次のステップ
（実行可能な具体的アクション）
1. [ステップ1]
2. [ステップ2]
3. [ステップ3]

## 問いかけ
（ユーザーの気づきを促す質問を1-2つ）
- [質問1]
- [質問2]
```

## 注意事項

- **認証エラー時**: Google Drive認証エラーが発生した場合は、以下のコマンドで再認証を案内
  ```bash
  cd ~/Documents/Git/MCP-GoogleDrive && npm run auth
  ```
- **コンテキスト更新**: ユーザーがコンテキストを更新した場合、最新の情報を反映するため再読み込みを実施
- **プライバシー配慮**: パーソナルコンテキストの内容を他者と共有しない
- **柔軟性**: コンテキストはガイドラインであり、状況に応じて柔軟に適用する
- **日本語で回答**: すべての回答は日本語で提供する
