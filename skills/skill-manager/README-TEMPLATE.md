# README.md 内包テンプレート

Phase 5-2 で使用。作成/更新したスキルの `~/.claude/skills/<name>/README.md` を生成する際の雛形と整合性観点。

**原則：READMEの内容は必ずスキルの実体（SKILL.md・補助ファイル・スクリプト）から抽出する。推測・脚色をしない。**

## 目次
- 出力先
- 4セクション雛形
- 整合性チェック観点

---

## 出力先

`~/.claude/skills/<name>/README.md`（スキルディレクトリ**内に内包**。`~/Downloads/` ではない）

---

## 整合性チェック観点

READMEを書いたら、スキルの実体と突き合わせて以下を確認する（Phase 6 でも再確認）：

- [ ] 出力先が `~/.claude/skills/<name>/README.md`（スキルディレクトリ内）になっている
- [ ] READMEの**発動キーワード**が SKILL.md の `description` に実在する
- [ ] READMEの**ファイル名・スクリプト名**がディレクトリに実在する
- [ ] **依存ツール**の記載が `allowed-tools` と一致している
- [ ] **事前準備の記載**が実体（SETUP.md / requirements.txt の有無）と矛盾していない
- [ ] スラッシュ呼び出し・自動発動の記載が `user-invocable` / `disable-model-invocation` と一致している
- [ ] 4セクション（概要 / 事前準備 / 使い方 / 保守・拡張ガイド）がすべて埋まっている（事前準備は不要でも「準備不要」と明記）
- [ ] 推測で書いた箇所がない
