# セルフレビュー用チェックリスト

Phase 6 で使用。作成/更新したスキルが公式ベストプラクティスに準拠しているか **網羅的に** 確認する。
不備があった項目は Phase 5 に戻って修正し、必須項目をすべて満たすまで繰り返す。

## コア品質（必須）
- [ ] `name` は小文字・数字・ハイフンのみ（最大64文字、予約語なし）
- [ ] `description` は「何をするか＋いつ使うか＋トリガー用語」を含む（最大1024文字）
- [ ] `description` は三人称で記述されている（一人称・二人称を使っていない）
- [ ] `description` に発動させたいキーワードが自然に含まれている
- [ ] `allowed-tools` は必要最小限の権限
- [ ] SKILL.md 本体は 500行未満
- [ ] 追加の詳細は別ファイルに分離されている（必要な場合）
- [ ] ファイル参照は 1レベルの深さ（深くネストしていない）
- [ ] 発動条件・使用タイミングが明確に記載されている
- [ ] 時間に依存する情報がない（または「古いパターン」セクションに隔離）
- [ ] 全体で一貫した用語を使用している
- [ ] 例が抽象的でなく具体的である
- [ ] ワークフローに明確なステップ（できればチェックリスト）がある
- [ ] progressive disclosure が適切に使われている
- [ ] 選択肢を出しすぎていない（デフォルトを提示）

## 配置・運用（プライベートスキル前提）
- [ ] `~/.claude/skills/<name>/` に配置されている
- [ ] ai-toolkit への master 作成・README 更新・同期を行っていない（[[feedback-private-skills]]）
- [ ] 100行超の参照ファイルには先頭に目次がある

## README.md 内包（必須）
- [ ] スキルディレクトリ内に `README.md` を内包している（出力先が `~/Downloads/` ではなく `~/.claude/skills/<name>/README.md`）
- [ ] 4セクション（概要 / 事前準備 / 使い方 / 保守・拡張ガイド）がすべて埋まっている（事前準備は不要でも「準備不要」と明記）
- [ ] README の発動キーワードが SKILL.md の `description` に実在する
- [ ] README のファイル名・スクリプト名がディレクトリに実在する
- [ ] README の依存ツール記載が `allowed-tools` と一致している
- [ ] スラッシュ呼び出し・自動発動の記載が `user-invocable` / `disable-model-invocation` と一致している
- [ ] 推測・脚色で書いた箇所がない（すべてスキルの実体に根拠がある）

## セキュリティ（必須）
- [ ] APIアクセストークン・APIキー・シークレット・パスワード等の機密情報を SKILL.md・参照md・スクリプト内に直書きしていない（[[feedback-no-hardcoded-secrets]]）
- [ ] 機密情報は環境変数・gitignore対象の設定ファイル・シークレットマネージャから読み込む形にしている
- [ ] md内のサンプルは実値でなくプレースホルダ（`<YOUR_TOKEN>` 等）を使っている

## 設定値の一元管理（該当する場合）
- [ ] CONFIG.md 等に定義する設定値を、参照ファイル（スクリプト・他のmd・手順）側で直書きしていない（[[feedback-config-single-source]]）
- [ ] CONFIG.md の値を変更すれば全体に反映される構造になっている（単一の情報源）
- [ ] スクリプトは CONFIG.md・環境変数・設定ファイルから値を取得している

## コードとスクリプト（該当する場合）
- [ ] スクリプトは問題を解決し、Claudeに丸投げしていない
- [ ] エラー処理が明示的で有用
- [ ] マジックナンバーがない（すべての値が正当化されている）
- [ ] 必要なパッケージが指示にリストされている
- [ ] スクリプトに明確なドキュメントがある
- [ ] Windowsスタイルのパスがない（すべてスラッシュ）
- [ ] 重要操作に検証/確認ステップがある
- [ ] MCPツールは完全修飾名（`ServerName:tool_name`）で参照している

## 仕上げ
- [ ] 実際の使用シナリオ（発動キーワード）でトリガーされそうか確認した
- [ ] 既存スキル（特に類似のもの）と役割が重複・競合していないか確認した
