# NotebookLM日次調査スキル（daily-notebooklm-research）

## 1. 概要

- **何をするスキルか**: 毎朝の定期実行用に、調査テーマを自動決定してWeb調査し、NotebookLM用のURLリストと音声解説プロンプトを生成する。
- **発動条件**:
  - 自動発動キーワード: なし（明示呼び出しのみ）
  - スラッシュ呼び出し: `/daily-notebooklm-research` で可能
  - 自動発動: なし（`disable-model-invocation: true`）
- **依存の有無**:
  - `scripts/read_doc.py`（本スキル内に内包。Googleドキュメント読み取り専用。他スキルは参照しない）
  - 過去セッションログの中間md（`~/Documents/ClaudeLogs/all-projects/`配下。存在しなくても動作する）
- **想定する利用シーン**: Claude Codeのルーチン（Cowork定期実行）機能で毎朝実行し、音声解説作成の前段（テーマ決定・ソース収集・プロンプト作成）を自動化する。

## 2. 事前準備（セットアップ）

- Googleドキュメントをフォールバック調査対象にする場合は、[SETUP.md](SETUP.md) の認証セットアップ（`~/.config/google-drive-skills/`への認証ファイル配置、`pip install -r requirements.txt`）が必要。
- 未調査テーマリストを常に使う運用にする場合は、追加の準備は不要。

## 3. 使い方

- **呼び出し方法**: `/daily-notebooklm-research` での明示呼び出し、またはCowork定期実行（毎朝）での自動起動
- **入力例**: 入力不要。`CONFIG.md` の「未調査テーマリスト」に事前にテーマを書いておくか、空のままにしてフォールバック調査に任せる
- **出力例**:
  - チャット上にURLリスト（コピペ用コードブロック）とNotebookLM用プロンプト（コピペ用コードブロック）を表示
  - `output/<YYYY-MM-DD>_<テーマ>.md` に同内容を保存
- **手順**:
  1. `CONFIG.md` の未調査テーマリストを確認（あれば先頭を採用）
  2. なければGoogleドキュメント・過去セッションログmd・ai-toolkit既存リソースの3方向から候補を洗い出し、総合的に比較してテーマを決定（`history.md` と重複しないものを選定）
  3. CONFIG.mdの「ソース選定時の心がけ」に従いWeb調査（3〜10件）
  4. URLリストとプロンプトを生成
  5. CONFIG.mdの心がけ項目に基づきセルフレビュー（最大3回まで修正）
  6. チャット表示＋ファイル保存
  7. `history.md` に追記、消費したテーマがあれば `CONFIG.md` のリストから削除
- **使う際のテクニック・コツ**:
  - 知りたいテーマが既に決まっている時は `CONFIG.md` の未調査テーマリストに書いておくと、フォールバック調査をスキップして即座にそのテーマで調査が始まる
  - セッションログmdは自動生成しない（トークン消費を避けるため）。最新の傾向を反映させたい場合は事前に該当する中間mdを用意しておく

## 4. 保守・拡張ガイド

### ファイル構成

| ファイル | 役割 |
|---------|------|
| SKILL.md | Phase構成本体（テーマ決定→調査→生成→セルフレビュー→保存→履歴更新） |
| CONFIG.md | 未調査テーマリスト・Googleドキュメント参照先・テーマ選定/ソース選定/プロンプト作成の心がけ・出力設定（人が直接編集） |
| SETUP.md | Googleドキュメント読み取りの認証セットアップ・トラブルシューティング |
| history.md | 調査済みテーマの履歴（日付・テーマ・出力ファイル・セルフレビュー試行回数）。スキルが自動追記 |
| scripts/auth.py | Google認証クライアントの取得（`read_doc.py`から呼び出される内部モジュール） |
| scripts/read_doc.py | Googleドキュメント1件の全文読み取りCLI |
| requirements.txt | scripts/が依存するPythonパッケージ |
| output/ | 生成したURLリスト・プロンプトの保存先 |

### 修正時の手順と注意点

- Phase 1（未調査テーマリスト）→Phase 2（フォールバック3方向の総合判断）という優先順位を変える場合はPhase 1・Phase 2の記述を両方更新する
- 「心がけ」項目を追加・変更する場合は、CONFIG.md本体とPhase 4.5のチェックリストの両方を同期させる（チェックリストが心がけ項目をそのまま転記する設計のため）
- `disable-model-invocation` を外すと自動発動するようになるが、Drive読み取りやWeb調査という副作用があるため非推奨

### 依存

- ツール: Read, Write, Bash, WebSearch, WebFetch, TodoWrite
- MCP / 外部: なし。他スキルへの参照もしない（Googleドキュメント読み取りは本スキル内の`scripts/read_doc.py`で自己完結）

### 動作確認・テスト方法

- `CONFIG.md` の未調査テーマリストに仮のテーマを1件追加し、`/daily-notebooklm-research` を実行する
- URLリストが3〜10件、プロンプトに焦点・ターゲット・フォーマット（基本「詳細」）・長さ（「デフォルト」固定）の指定が含まれているかを確認する
- 実行後、`history.md` に1行追記され、`CONFIG.md` から消費したテーマが消えていることを確認する
