# 指示
1. 引数として指定されたGitHub Issueのリンクが記載されたmdファイル（$1）を読み取り、MCPによりIssue情報を取得する。 
2. 説明や仕様に不明点がある場合はユーザーに質問して内容を明確化する。  
3. すべてが明確になったら、プロジェクトの関連コードを読み込む。  
4. 以下の項目を含むレポートをMarkdown形式で生成する：  
    - 概要
    - ストーリーポイント（見積理由付き）  
    - 実装方針  
    - 変更が必要なファイルと具体的な変更内容  

※日本語で回答すること
