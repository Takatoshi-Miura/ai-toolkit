# 問題解決: リスク計画（Risk）

## 概要
問題解決プロセスのPhase 7を実行する。リスクを洗い出し、予防策・発生時対策を検討する。

## 前提条件
- Phase 5-6（タスク分解・スケジュール作成）が完了していること
- 実行スケジュールが作成されていること

## 手順

### Phase 7: リスク計画（Risk）

1. 「『リスク計画（Risk）：リスクを洗い出し、予防策・発生時対策を検討する』フェーズに入ります。」とユーザーに伝える
2. 問題解決プロセス「リスク計画（Risk）：リスクを洗い出し、予防策・発生時対策を検討する」に基づき、情報を整理する。<br>不足している情報や不明点については、不明点がなくなるまでユーザに質問して解決する。
3. 「リスク計画（Risk）のフォーマット」の形式で情報を整理してユーザーに了承を得る（チェックリストも活用する）。了承が得られたらmdファイルに追記する。
4. 了承が得られない場合、得られるまで2,3を繰り返す。

## 問題解決プロセス

### リスク計画（Risk）：リスクを洗い出し、予防策・発生時対策を検討する

#### Step1: リスクの洗い出し
- マイルストーン遅れは全体の遅れに直結するため、マイルストーンに悪影響を及ぼす事象から優先的に検討する
- 4要素（品質、範囲、時間、リソース）に悪影響を与え得ることを網羅的に検討する
- 範囲・品質リスクを洗い出すためのヒント：
    - 代替手段が乏しいタスクはないか（コスト・時間投下での挽回が困難）
    - KGI・KPIに悪影響を与えるタスクはないか
    - それが起こると全て台無しになるリスクはないか

#### Step2: リスクの重みづけ
- 洗い出した各事象について、以下のスコアをつける
    - 発生確率：どのくらいの頻度で起こりそうか（大・小 の2段階評価）
    - 影響：発生した場合の影響の大きさ（大・小 の2段階評価）
- 評価に応じて分類する
    1. 発生確率：大、影響：大 → 予防策／発生時対策ともに検討する
    2. 発生確率：大、影響：小 → 1を検討した上で余裕があれば検討する
    3. 発生確率：小、影響：大 → 発生時対策を優先して検討する
    4. 発生確率：小、影響：小 → 対策検討の必要なし

#### Step3: 予防策／発生時対策の検討
- 洗い出したリスクに対して、以下を検討する
- 予防策とは、リスク事象の発生確率を減らす対策のこと
- 発生時対策とは、リスク事象が発生した場合の影響を軽減する対策のこと。関係者と事前に練っておくことで有効性が高まる
- トリガーポイントとは、発生時対策を実行する条件のこと

#### リスク計画（Risk）のチェックリスト
- [] マイルストーンに悪影響を及ぼす事象を優先的に検討できているか
- [] 4要素（品質、範囲、時間、リソース）の観点でリスクを網羅できているか
- [] 各リスクに発生確率と影響の評価がついているか
- [] 評価に応じた適切な対策（予防策・発生時対策）が検討されているか
- [] トリガーポイントが明確に定義されているか
