# AI Toolkit

Claude CodeのカスタムスラッシュコマンドやAI活用のためのリソースをまとめたリポジトリです。

## ディレクトリ構成

```
ai-toolkit/
├── .github/
│   └── workflows/     <- GitHub Actions ワークフロー定義
├── slash-commands/     <- Claude Codeのカスタムスラッシュコマンド用
├── agents/            <- エージェント関連の定義や設定
├── rules/             <- ルールファイルやポリシー、ワークフロー
├── templates/         <- 定型テンプレートファイル、応答フォーマットなど
├── output-style/      <- Claude Codeの出力スタイル設定
├── scripts/           <- 自動化スクリプト（Python等）
├── task/              <- タスク定義ファイル
└── README.md          <- このファイル
```

## 各ディレクトリの目的

### slash-commands/
Claude Code用のカスタムスラッシュコマンドを保存します。開発作業を効率化するための独自コマンドを管理。

### agents/
AI エージェントの設定や定義ファイルを保存します。特定のタスクに特化したエージェントの構成を管理。

### rules/
プロジェクトのルールファイル、ポリシー、ワークフローを保存します。コーディング規約やレビュープロセスなどを定義。

### templates/
定型的なテンプレートファイルや応答フォーマットを保存します。よく使用するコードのテンプレートやドキュメントの雛形を管理。

### output-style/
Claude Codeの出力スタイル設定ファイルを保存します。レスポンスの形式や表示方法をカスタマイズするためのスタイル定義を管理。

### .github/workflows/
GitHub Actions のワークフロー定義ファイルを保存します。定期実行やCI/CDなどの自動化処理を管理。

### scripts/
自動化に使用する Python スクリプトなどを保存します。GitHub Actions から呼び出されるスクリプトを管理。

### task/
タスク定義ファイルを保存します。スラッシュコマンドから参照されるタスクの詳細な手順を管理。

## 使用方法

1. 各ディレクトリに適切なファイルを配置
2. Claude Codeでスラッシュコマンドを使用してファイルを参照
3. 必要に応じてテンプレートやルールをプロジェクトに適用

## 目的

今後のAI活用において、効率的な開発環境を構築し、再利用可能なリソースを体系的に管理することを目的としています。

## 🤖 GitHub Actions 自動化

このリポジトリには、GitHub Actions を使った自動化ワークフローが含まれています。

### 📰 毎朝のニュース自動取得 💰 完全無料

毎朝7時（JST）に Yahoo ニュースを自動取得し、GitHub Issue として作成します。

**✨ 完全無料版**：BeautifulSoup を使用した HTML パース方式のため、API キー不要でコスト0円で運用できます！

#### セットアップ手順

1. **ワークフローを有効化**
   - リポジトリの "Actions" タブを開く
   - "Daily Yahoo News Fetcher" ワークフローを選択
   - "Enable workflow" をクリック（初回のみ）

2. **手動実行でテスト（推奨）**
   ```bash
   # Actions タブ > Daily Yahoo News Fetcher を選択
   # "Run workflow" ボタンをクリック
   ```

これだけ！API キーの設定は不要です 🎉

#### ワークフローの詳細

- **実行スケジュール**: 毎朝 7:00 JST（22:00 UTC 前日）
- **取得内容**: Yahoo ニュースの国内・国際・経済カテゴリから各8件
- **出力先**: GitHub Issue に自動作成（`news` と `automated` ラベル付き）
- **技術スタック**: Python + BeautifulSoup4 + Requests
- **コスト**: 完全無料 💰 $0

#### コストについて

- **API 使用料**: $0（Claude API 不使用）
- **GitHub Actions**: パブリックリポジトリなら無料、プライベートでも月2000分まで無料
- **実行時間**: 約1-2分/日 → 月間で約30-60分程度
- **合計コスト**: 完全無料 🎉

#### カスタマイズ

実行時刻やカテゴリをカスタマイズする場合は、以下のファイルを編集してください：

- `.github/workflows/daily-news.yml` - スケジュール設定
- `scripts/fetch_yahoo_news.py` - 取得カテゴリやフォーマット

#### ⚠️ 注意事項

- Yahoo ニュースの HTML 構造が変更された場合、記事の取得に失敗する可能性があります
- その場合は `scripts/fetch_yahoo_news.py` の `extract_news_from_html()` 関数を修正する必要があります
- スクレイピングは Yahoo の利用規約に従って行ってください
- 過度なリクエストは行わないでください（現在は1日1回のみ実行）
